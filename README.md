# E-Commerce Funnel & Revenue Optimization Dashboard

## Project Overview
Built an end-to-end E-commerce Conversion Funnel Analysis Dashboard to identify user drop-offs, evaluate traffic source performance, and uncover revenue optimization opportunities using SQL and data visualization.

---

## Objective
The goal of this project was to analyze customer behavior across the sales funnel and identify:
- Where users drop off before purchase
- Which traffic sources generate the highest revenue
- Which marketing channels deliver the best conversion rates

---

## Tools & Technologies
- MySQL
- Power BI
- Data Cleaning & Aggregation
- Funnel Analysis
- KPI Reporting

---

## Dataset Information
The dataset contains 30 days of e-commerce user interaction data including:
- User IDs
- Funnel Events (`page_view`, `add_to_cart`, `checkout`, `payment`, `purchase`)
- Transaction Amount
- Traffic Sources (`organic`, `paid_ads`, `email`, `social`)
- Event Timestamps

---

## Key KPIs
- **Total Revenue:** 87.98K
- **Average Order Value (AOV):** 106.51
- **Revenue per Visitor:** 17.60
- Funnel Conversion Rates
- Revenue by Traffic Source
- User Drop-off Analysis

---

## Dashboard Features
- User Funnel Visualization (Views → Purchase)
- Conversion Rate by Traffic Source
- Revenue by Traffic Source
- Views by Traffic Source
- KPI Summary Cards

---

## Key Insights
- Highest user drop-off occurs between **page view** and **add to cart**
- **Email traffic** had the highest conversion rate (34%)
- **Organic traffic** generated the highest revenue (~37K)
- **Social media** brought high traffic but the lowest conversion rate (7%)
- Checkout-to-purchase conversion remained highly optimized

---

## Business Recommendations
- Improve product pages and user engagement at the top of the funnel
- Increase focus on high-converting channels like email marketing
- Reduce spend on low-performing social campaigns
- Optimize acquisition strategy based on revenue contribution instead of traffic volume
